Tell me about yourself and why you're interested in this role.
Easy
⌄
Model Answer
Start with your current role and a key achievement, then work backwards through your career. For example: "I'm a Senior Software Engineer with 4 years building high-scale distributed systems. At Meta, I led a team that reduced API latency by 40% for 2 billion users. This role excites me because your Kubernetes infrastructure work aligns directly with what I've been building toward."
⚡ Pro Tip
Keep to 90 seconds. End with why THIS specific company excites you — show you've researched them.
2
How would you design a real-time notification system at scale?
Hard
⌄
Model Answer — System Design
1. Clarify requirements — scale, latency, delivery guarantees.

2. Core components — Kafka for message queue, WebSocket servers, user presence service via Redis, push notification gateway.

3. Scaling — Horizontal scaling, partition by user ID, Redis for sessions, CDN for assets.

4. Edge cases — Offline delivery, deduplication with idempotency keys, retry with exponential backoff.
⚡ Your Experience
Reference your Meta notification system work: "At Meta, I built exactly this — handling 50M+ daily notifications using..."
3
Describe a time you led a team through a difficult challenge.
Medium
⌄
Model Answer — STAR Format
S: At Meta, my team needed to migrate 12 legacy services to Kubernetes in 6 weeks with zero downtime.

T: As tech lead, I coordinated 4 engineers and maintained 99.9% uptime throughout.

A: Created a phased migration plan, built shadow traffic validation, ran daily 15-min standups.

R: Done in 5 weeks. Deployment time: 2 hours → 8 minutes. Zero downtime incidents.
⚡ Always Quantify
Numbers tell the story. "93% reduction in deployment time" beats "much faster deployments."
4
When would you use SQL vs NoSQL? Give real examples.
Medium
⌄
Model Answer
SQL (PostgreSQL, MySQL): ACID, complex joins, strong consistency. Best for: financial data, user accounts, anything needing referential integrity.

NoSQL (MongoDB, Cassandra, DynamoDB): Horizontal scale, flexible schema. Best for: activity feeds, product catalogs, session data, event logs.

My rule: "Complex multi-join queries → SQL. High write throughput or variable schema → NoSQL."
⚡ Bonus Points
Mention NewSQL (CockroachDB, Spanner) — shows you're current with distributed data systems.
5
How do you handle disagreement with a senior engineer?
Medium
⌄
Model Answer
"I make sure I fully understand their perspective first, then share my concern with data — not opinions. If we still disagree after a genuine discussion, I defer to their decision while documenting my concern, and ask to revisit if results don't match expectations."
⚡ Red Flags to Avoid
Don't say "I always defer" (no backbone) or "I push back hard" (difficult to work with). The middle ground wins.
6
Walk me through debugging a live production outage.
Hard
⌄
Model Answer — 5-Step Protocol
1. Contain: Roll back or feature-flag off. Protect users first.

2. Diagnose: Check recent deploys, error spikes in Grafana/Datadog, first error in logs.

3. Hypothesize: 2–3 causes ranked by probability, starting with most recent change.

4. Verify: Test each systematically. Never make multiple changes at once.

5. Fix + Post-mortem: Deploy, monitor, blameless post-mortem within 48 hours.
⚡ Reference Real Experience
Reference a real outage: "We restored 99.9% uptime in 12 minutes by finding a misconfigured load balancer timeout."
